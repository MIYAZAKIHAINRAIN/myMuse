# myMuse 収益配分・役割分担提案書

**作成日**: 2026年3月1日  
**作成者**: 開発担当  
**対象**: RATIO Lab. 様

---

## 1. はじめに

本書は、小説執筆支援AIアプリ「myMuse」の収益配分および役割分担について、明確な合意形成を目的として作成しました。

---

## 2. 提案：収益配分

### 基本配分

| 項目 | 開発者 | RATIO Lab. |
|------|--------|------------|
| **売上配分** | 50% | 50% |

### 配分の根拠

- **開発者**: 企画・設計・実装・保守・技術サポート
- **RATIO Lab.**: 事業運営・顧客対応・マーケティング・経理・法務

両者の貢献度は同等であり、対等なパートナーシップとして50:50を提案します。

---

## 3. 役割分担

### 3.1 開発者の担当業務

| カテゴリ | 業務内容 | 想定工数 |
|----------|----------|----------|
| **開発** | 新機能の企画・実装 | 週5〜10時間 |
| **保守** | バグ修正、パフォーマンス改善 | 随時 |
| **技術サポート** | 技術的問い合わせへの回答（RATIO Lab.経由） | 週1〜2件 |
| **監視** | エラーログ確認、システム異常検知 | 週1〜2回 |
| **デプロイ** | 本番環境への更新反映 | 更新時 |
| **ドキュメント** | 技術仕様書・操作マニュアル作成 | 随時 |

### 3.2 RATIO Lab. の担当業務

| カテゴリ | 業務内容 | 想定工数 |
|----------|----------|----------|
| **顧客対応** | ユーザーからの問い合わせ一次対応 | 随時 |
| **運営管理** | 管理者ダッシュボードでの売上・ユーザー確認 | 毎日5〜10分 |
| **マーケティング** | 招待コード作成、キャンペーン企画・実施 | キャンペーン時 |
| **ユーザー管理** | クレジット付与、プレミアム設定 | 問い合わせ対応時 |
| **経理** | 売上管理、入金確認、税務処理 | 月次 |
| **法務** | 利用規約・プライバシーポリシー管理 | 随時 |
| **返金処理** | KOMOJUダッシュボードでの返金対応 | 発生時 |

## 4. 管理者ダッシュボード機能

開発者が構築した管理者ダッシュボード（`/admin`）により、RATIO Lab.は以下の業務を自立して実行できます。

### 4.1 実装済み機能

| 機能 | 説明 | 操作方法 |
|------|------|----------|
| **売上統計** | 今日/今週/今月の売上をリアルタイム表示 | ダッシュボード確認 |
| **ユーザー統計** | 総ユーザー数、プレミアム数、新規登録数 | ダッシュボード確認 |
| **作品統計** | 総作品数、総文字数 | ダッシュボード確認 |
| **AI利用量** | 消費クレジット、残高合計 | ダッシュボード確認 |
| **決済履歴** | 最近の決済一覧（50件） | 「決済履歴」タブ |
| **ユーザー管理** | 検索、クレジット付与、プレミアム切替 | 「ユーザー管理」タブ |
| **招待コード管理** | 作成、有効/無効切替、使用状況確認 | 「招待コード」タブ |
| **データエクスポート** | 全データ/個別ユーザーのJSON出力 | 「エクスポート」タブ |
| **操作ログ** | 管理者の全操作履歴（監査証跡） | 「ログ」タブ |
| **エラーログ** | システムエラーの記録・確認 | 「ログ」タブ |

### 4.2 効率化の効果

| 作業 | 従来の方法 | 現在 | 時間削減 |
|------|------------|------|----------|
| 売上確認 | データベース直接操作（10分） | ダッシュボード確認（10秒） | **98%** |
| ユーザー検索 | データベース直接操作（5分） | 検索ボックス（10秒） | **97%** |
| クレジット付与 | 開発者へ依頼（1日） | ボタン操作（30秒） | **99%** |
| 招待コード作成 | 開発者へ依頼（1日） | フォーム入力（1分） | **99%** |
| データ出力 | 開発者へ依頼（数日） | ボタン操作（5秒） | **99%** |

**月間削減時間（推定）**: 約10〜15時間 → RATIO Lab.が自立運営可能

## 6. 経費の取り扱い

### 6.1 開発者負担

| 項目 | 月額目安 |
|------|----------|
| Cloudflare（インフラ） | $0〜20 |
| AI API（Grok/Gemini） | 使用量による |
| ドメイン（年額按分） | 約¥100 |

### 6.2 RATIO Lab. 負担

| 項目 | 月額目安 |
|------|----------|
| KOMOJU決済手数料 | 売上の3.6% |
| 会計・税務 | 既存経費内 |

### 6.3 売上からの控除

```
売上総額
  - KOMOJU手数料（3.6%）
  - AI API使用料（実費）
  ─────────────────────
  = 純利益
  
純利益 × 50% = 開発者取り分
純利益 × 50% = RATIO Lab.取り分
```
