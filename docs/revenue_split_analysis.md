## 50:50配分のメリット・デメリット

### メリット
- シンプルで分かりやすい
- 両者が対等なパートナーシップ
- 恩師との良好な関係維持
- 販売側のモチベーション向上

### デメリット
- 開発・保守の労力に対して低い可能性
- 運用コストを考慮すると実質的に開発者の取り分が減る
- 将来的な開発投資の資金が限られる

### 検討すべき追加条件（50:50にする場合）
1. **運用コストの負担**: 決済手数料・API費用は売上から差し引いてから配分
2. **サポート分担**: 技術サポートは開発者、営業サポートは恩師会社
3. **独占権の有無**: 恩師会社のみが販売できるか
4. **契約期間**: 1年ごとに見直し
5. **最低保証**: 恩師会社が月間○件以上の販売を目標とする

## 推奨される契約形態

### 案1: 純利益ベースの50:50
```
配分対象 = 総売上 - 決済手数料 - 運用コスト
開発者取り分 = 配分対象 × 50%
恩師会社取り分 = 配分対象 × 50%
```

### 案2: 総売上ベースの50:50（運用コストは開発者負担）
```
開発者取り分 = 総売上 × 50% - 決済手数料 - 運用コスト
恩師会社取り分 = 総売上 × 50%
```
※ この場合、実質的に開発者は40-45%程度になる

### 案3: 段階的配分
```
月売上 〜10万円: 開発者60% / 恩師会社40%
月売上 10万〜50万円: 開発者55% / 恩師会社45%
月売上 50万円〜: 開発者50% / 恩師会社50%
```

## まとめ

50:50の配分は、恩師との関係を重視する場合に適した選択です。ただし、以下の点を契約書に明記することを推奨します：

1. **費用負担の明確化**: 運用コストをどちらが負担するか
2. **役割分担の明確化**: 開発・保守 vs 営業・販売
3. **見直し条項**: 売上や状況に応じて配分を見直せる条項
4. **撤退条件**: どちらかが撤退する場合のルール
